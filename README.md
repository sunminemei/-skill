 技能名称: vehicle-dtc-diagnosis（车辆故障诊断交互式助手）v1.0.0

  ---
  一、角色与行为规则
  - 以诊断技师角色工作
  - 会话开始：询问车辆信息（车型、VIN、故障现象）、确认诊断工具
  - 逐步诊断：一个系统一个系统排查，等用户反馈后解读
  - 结果记录：每步记录ECU诊断ID、DTC码、故障描述、判定结论
  - 完成后输出标准诊断报告

  二、诊断域菜单（10项）
  1. 全车DTC扫描
  2. 动力系统（发动机/变速箱/混动）
  3. 底盘系统（ABS/ESC/转向/悬架）
  4. 车身系统（BCM/灯光/门窗/座椅）
  5. 网络通信（网关/各网段通信状态）
  6. 电池/高压系统（BMS/OBC/VCU，新能源车）
  7. ADAS/自动驾驶（雷达/摄像头/域控）
  8. 信息娱乐（车机/仪表/音响）
  9. 清除故障码
  10. 生成诊断报告

  三、UDS诊断服务速查表（9个服务：10/11/14/19/22/27/2E/3E，含功能与子功能说明）

  六、DTC故障码解析规则（ISO 15031-6）
  - 3字节编码规则：类别(高2位) + 系统 → P/C/B/U码分类
  - 含解析示例（P0501、P1186、P0010）

  七、ECU诊断ID映射表
  - 40+个ECU的诊断ID/应答ID/名称/所属系统映射（覆盖动力、底盘、车身、信息四大域）

  八、分系统诊断方法（6个子系统）
  - 6.1 动力系统：14步诊断流程（ECU: 0x704/0x70F/0x723/0x775-777）
  - 6.2 底盘系统：9步诊断流程（ECU: 0x716/0x717/0x758/0x75A-C）
  - 6.3 车身系统：7步诊断流程（ECU: 0x71A/0x718/0x71C等）
  - 6.4 网络通信：7步诊断流程（ECU: 0x709网关）
  - 6.5 高压系统：10步诊断流程（ECU: 0x723/0x724/0x72E等）
  - 6.6 ADAS系统：6步诊断流程（ECU: 0x730/0x732-737等）

  九、DTC状态字节解读
  - 8位状态位含义表 + 3种常见值解读（0x2F立即维修 / 0x0B故障持续 / 0x08历史故障）

  十、诊断报告模板
  - 标准化4段式报告：DTC汇总 → 系统分类诊断 → 维修优先级 → 签章

  十一、安全注意事项
  - 6条安全规则（高压安全、气囊安全、制动安全、ADAS标定、不随意清码、DTC不等于换件）
